# 객체지향 프로그래밍 (OOP) 정리

## 1. 객체지향이란?
객체를 중심으로 프로그램을 설계하는 방식

+ 현실 세계를 코드로 모델링

+ 데이터 + 기능을 하나로 묶음

---

## 2. 추상화 (Abstraction)
필요한 정보만 남기고 불필요한 부분은 숨긴다.

+ 핵심 기능만 표현

+ 인터페이스 / 추상 클래스 사용

---

## 3. 상속 (Inheritance)
기존 클래스를 재사용하여 새로운 클래스를 만든다.

+ 부모 클래스 → 자식 클래스

+ 코드 재사용 가능

---

## 4. 다형성 (Polymorphism)
하나의 객체가 여러 형태로 동작할 수 있다.

+ 같은 메서드, 다른 동작

+ 오버라이딩 / 오버로딩

---

## 5. 캡슐화 (Encapsulation)
데이터와 기능을 하나로 묶고 외부 접근을 제한한다.

+ private 사용

+ getter / setter 활용

---

## 6. 객체지향의 장점
+ 유지보수 용이

+ 코드 재사용성 증가

+ 확장성 향상

---

## 정리
객체지향은  
코드를 구조적으로 관리하고 확장하기 위한 핵심 개념이다.

###### 이후 클래스, 객체 설계 능력의 기반이 된다.
