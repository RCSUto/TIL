# HTTP 정리

## 1. HTTP란?
HTTP(HyperText Transfer Protocol)는  
클라이언트와 서버가 데이터를 주고받기 위한 통신 규약이다.

+ 웹의 기본 통신 방식

+ 요청(Request) → 응답(Response) 구조

---

## 2. HTTP Method
클라이언트가 서버에 요청할 때 사용하는 방식

+ GET : 데이터 조회

+ POST : 데이터 생성

+ PUT : 데이터 전체 수정

+ PATCH : 데이터 일부 수정

+ DELETE : 데이터 삭제

---

## 3. HTTP Status Code
서버의 응답 상태를 나타내는 코드

### 성공 (2xx)
+ 200 : 요청 성공

+ 201 : 생성 성공

### 클라이언트 오류 (4xx)
+ 400 : 잘못된 요청

+ 401 : 인증 필요

+ 403 : 접근 금지

+ 404 : 리소스 없음

### 서버 오류 (5xx)
+ 500 : 서버 내부 오류

---

## 4. HTTP의 무상태성 (Stateless)
HTTP는 상태를 저장하지 않는다.

+ 각 요청은 서로 독립적이다

+ 이전 요청의 정보를 기억하지 않는다

### 특징
+ 서버 확장에 유리

+ 요청 처리 단순

### 문제점
+ 로그인 상태 유지 어려움

### 해결 방법
+ 쿠키 (Cookie)

+ 세션 (Session)

+ 토큰 (JWT)

---

## 5. HTTPS

HTTP에 보안이 추가된 프로토콜

- HTTP + SSL/TLS 암호화

### 특징
+ 데이터 암호화

+ 도청 및 위변조 방지

+ 서버 신뢰성 확보

### 비교
+ HTTP : 평문 전송 (보안 취약)

+ HTTPS : 암호화 전송 (보안 안전)
